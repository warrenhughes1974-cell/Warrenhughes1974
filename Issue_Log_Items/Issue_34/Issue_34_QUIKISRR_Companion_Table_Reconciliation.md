## F. Companion table gap analysis

1. **QuikClms:** the claims pipeline intentionally emitted only death, full-surrender, and 0090-disbursement chains. Partial surrenders (0561) were never in claims emit scope — 0 of 3,623 events have a claim row. If QLAdmin requires one claim (with incrementing sequence, per SME comment 3) per historical partial withdrawal, **~3,623 QuikClms rows and their QuikClmp payments would need to be created** — more than doubling the current claims table (2,114 rows).
2. **QuikClmp:** payments exist only for the emitted claim families; nothing to pair 0561 events to.
3. **QuikBene/QuikBenh:** no emit exists in this repo, and the client's Loyal2QL converter notes claim quikbenh is populated **on the client side** ("Benefit / dividend history — done"). Whether client-side QuikBenh already carries partial-withdrawal history (benefit code 8) **cannot be verified from this workspace** — it requires a client-side extract or SME confirmation.
4. **Claim sequence:** `MSEQ = 0` across quikclms; the SME-described per-withdrawal sequence does not exist in any converted output.
5. **Transaction audit:** no converted audit table exists in this pipeline.

## I. Final recommendation

## **BLOCKED**

The reconciliation proves the companion record set **does not exist** for the ISWL 0561 population: zero event-level matches across QuikClms/QuikClmp, no QuikBene/QuikBenh output in this repo, no per-withdrawal claim sequence anywhere. Q11 is therefore a genuine scope decision only the SME can make, and its answer changes the size of PR-7 (QuikIsrr-only ≈ one policy-table emit; full record set ≈ doubling the claims tables plus sequencing and client-side QuikBenh coordination).

**Path to unblock:** SME answers Q11 (Option 1 vs Option 2, plus client-side QuikBenh confirmation). If Option 1 (QuikIsrr-only) is chosen, Issue #34 reverts to **READY AFTER SME CONFIRMATION** with only Q7 (MISWL), Q10 (output path), and the standing sign-offs outstanding.

The QuikIsrr business rule itself (Decision Review §H — 3,623 rows / 636 policies / $1,217,593.55) required **no regeneration**: this reconciliation independently rebuilt the eligible set and matched the expected population exactly.

Do **not** create QuikIsrr.csv, loaders, emit code, or begin PR-7 until Q11 is answered and documented.
