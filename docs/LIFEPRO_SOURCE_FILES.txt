================================================================================
LIFEPRO SOURCE FILES — QLA Migration Conversion
================================================================================
Platform:     LifePRO -> QLAdmin Enterprise Migration (app.py)
Engine:       v57.24+
Maintained:   Update this file whenever qla_core/lifepro_source_resolver.py changes
Location:     docs/LIFEPRO_SOURCE_FILES.txt

WORKFLOW (no manual renames required)
--------------------------------------
1. Unzip LifePRO monthly extracts into QLA_Migration/Source/  (flat folder)
2. Copy aba_routing_lookup.csv if not already present (see Derived section)
3. Run conversion batch (run_converter.bat or EXECUTE FULL BATCH MIGRATION)

The engine accepts standard LifePRO export filenames. Legacy project aliases
(quikmstr.csv, PPBEN.csv, etc.) still work as fallbacks if LifePRO names are
not present.

LEGACY PROJECT ALIASES (rollback fallback — do not use for new loads)
================================================================================

If LifePRO-named files are absent, the engine falls back to:

  quikmstr.csv                     (was renamed PPOLC)
  PPBEN.csv, PPBENTYP.csv          (short LifePRO names)
  quikclid.csv / quikbenf.csv      (copies of RNA — no longer needed)

================================================================================
NOT VALID SOURCE INPUT
================================================================================

Do NOT place QLAdmin conversion OUTPUT in Source/:
  - Output/quikclnt.csv with MCLIENTID columns (already converted)
  - Any file from QLA_Migration/Output/ used as LifePRO input

================================================================================
CLAIMS UAT (separate from LifePRO Source/)
================================================================================

Claims tables (quikclms, quikclmp) are emitted from pre-built candidate files
under claims_analysis/ (Phase 24 post-rebalance preferred). Not LifePRO extracts.
