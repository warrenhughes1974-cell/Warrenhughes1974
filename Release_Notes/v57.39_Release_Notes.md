## Summary

Release **v57.39** stops LifePRO benefit type **SL (Substandard Life)** from emitting as separate `quikridr` coverage rows. SL is substandard rating metadata, not an additional death benefit; QLAdmin handles substandard life through its product rating structure.

Also included in this release line: **#21D** (ISWL MDEPINT 4.50, RNA blank-name B1), **#21J rollback** (removed `[CONVERSION]` modal factor memos), and cumulative preservation of **#25, #26, #28, #21M, #21M-FU, #21K** fixes.

## Issue #21K — PUA precision (CLOSED)

Issue **#21K** is **closed** with no converter changes at v57.39.

| Layer | Status |
|-------|--------|
| Conversion (`quikridr.csv` MUNIT) | **Correct** — five-decimal units preserved (e.g. `010448806C` PUA: `5.75296`) |
| DBF storage | **N(10,5)** field precision update complete |
| QLAdmin Coverage tab (Amount Ins) | **Display-only** whole-dollar rounding (e.g. `$5,753.00` vs true face `$5,752.96`) |
| Death benefit / payment | Uses stored five-decimal units — **pays correctly** |

**Client note:** Coverage screen Amount Ins is not the authoritative payment value; stored `MUNIT × MVPU` precision is preserved through conversion and benefit calculation.
