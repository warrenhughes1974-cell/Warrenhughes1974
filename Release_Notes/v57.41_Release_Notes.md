## Summary

Release **v57.41** wires all approved ISWL workstreams into the **full batch migration** path so operators can run one client UAT conversion from the GUI:

| Workstream | Tables / outputs | Batch hook |
|------------|------------------|------------|
| Issue #31 | QuikCvs, QuikGps, QuikCoi, QuikGcoi (+ keys) | R5 rate loader subprocess |
| Issue #32 | quikloan.csv | Env-gated table emit in batch loop |
| Issue #32 Phase 5 | QuikUint.csv | R5 rate loader (GUI runner fix) |
| Issue #33 Phase 6 | QuikIssc.csv | R5 rate loader (GUI runner fix) |
| Issue #34 PR-7 | quikclms/clmp append, quikbenh, QuikIsrr | Post-claims batch finale subprocess |

**UAT launcher** (`QLA_Migration/run_converter.bat`) sets all required env flags and launches `python app.py` from repo root.

---

## Environment flags (UAT full batch)

| Variable | Purpose |
|----------|---------|
| `QLA_RUN_MODE=UAT` | UAT claims orchestration |
| `QLA_BATCH_INCLUDE_CLAIMS_UAT=1` | Emit quikclms/quikclmp after standard tables |
| `QLA_ENABLE_QUIKLOAN_EMIT=1` | Run QuikLoan in batch loop |
| `QLA_QUIKLOAN_WRITE_OUTPUT=1` | Write quikloan.csv to Output |
| `QLA_BATCH_INCLUDE_RATE_TABLES=1` | Rate tables at batch finale |
| `QLA_ENABLE_QUIKISRR_EMIT=1` | QuikIsrr package after claims emit |

Production / partial runs: omit flags to preserve prior gated behavior.

## Operator steps (client UAT)

1. Run `QLA_Migration\run_converter.bat` (or set env flags manually).
2. Confirm paths per RUN_GUIDE (Source, Output, rulebook, crosswalk).
3. Run **Product Setup** if `quikplan.csv` is not already in Output.
4. Click **EXECUTE FULL BATCH MIGRATION**.
5. Deliver `QLA_Migration/Output/` (table CSVs + `rates/` subfolder) for QLAdmin load.
