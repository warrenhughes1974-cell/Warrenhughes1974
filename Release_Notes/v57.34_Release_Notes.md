## Major Enhancements

### Issue #21M / #21M-FU — Policy Notes & ENS → QUIKMEMO

- New greenfield pipeline converts LifePRO **PNOTE** and **PENSE** to QLAdmin **QUIKMEMO**.
- Dual-source merge: 6,003 PNOTE segments + 23,276 PENSE segments → **4,380 rows** (one per policy key).
- Segment text preserved with `[PNOTE]` / `[ENS]` headers; multi-segment policies joined with `\n---\n`.
- DBF + DBT packaged co-located in `QLA_Migration/Output/quikmemo_uat_dbf/` (hygiene skip for `*_uat_dbf/` dirs).
- **MEMOKEY** uses Issue #25 10-char left-pad (`format_qladmin_mpolicy()`).

### Cumulative (preserved from prior releases)

- **#25:** All MPOLICY fields exactly 10 characters across 279,222 checked values.
- **#26:** MPREM populated on all 7,002 quikridr rows; modal premium on quikmstr unchanged.
- **#21B/C/H:** Rulebook and engine fixes for bill day, policy fees, and 9-digit ABA routing.

## Deployment Notes

### Engine deploy

1. Deploy updated `app.py` (or `QLA_Migration/app.py` bundle) at **v57.34**.
2. Deploy `qla_core/quikmemo_converter.py`, `qla_core/quikmemo_dbf_generator.py`, `qla_core/run_logging.py`, `qla_core/normalize_utils.py`.
3. Deploy updated rulebooks: `Sync_Rulebook_quikridr.csv`, `Sync_Rulebook_quikplan.csv`, `Sync_Rulebook_quikmstr.csv` (as applicable).
4. Run full batch: `QLA_Migration/run_converter.bat` or `_run_full_batch_test.py`.

### QUIKMEMO DBF deploy

Copy **both** files together from:

